# ก้าวต่อไปสำหรับการพัฒนา UI Bot (Future Plans) 🚀

เอกสารนี้รวบรวมไอเดียและแผนงานในอนาคต สำหรับการนำบอททดสอบเว็บศาลรัฐธรรมนูญไปต่อยอดให้เก่งขึ้นในระดับ Enterprise

## 1. เพิ่มเคสทดสอบหน้าอื่นๆ (Add More Test Cases) 📝
- **แนวคิด:** ขยายขอบเขตการทดสอบจากการค้นหา ไปยังระบบอื่นๆ ที่สำคัญของเว็บไซต์
- **สิ่งที่ต้องทำ:**
  - เพิ่มเมธอดใน `pages/home_page.py` เพื่อจำลองการกดเข้าเมนูต่างๆ (เช่น "คำวินิจฉัยศาล", "ข่าวประชาสัมพันธ์")
  - ทดสอบระบบการเปลี่ยนภาษา (ไทย/English) ว่าทำงานได้ปกติ
  - ทดสอบการดาวน์โหลดไฟล์เอกสาร (PDF) ว่าดาวน์โหลดได้สำเร็จและไฟล์ไม่เสีย

## 2. แจ้งเตือนเฉพาะตอนที่เว็บพัง (Alert on Failure Only) 🚨
- **แนวคิด:** ลดความน่ารำคาญของการแจ้งเตือนทุกวัน โดยให้บอทส่งข้อความเข้า LINE เฉพาะเมื่อตรวจพบปัญหา
- **สิ่งที่ต้องทำ:**
  - แก้ไขไฟล์ `conftest.py` ในฟังก์ชัน `pytest_sessionfinish`
  - เพิ่มเงื่อนไข `if failed > 0 or errors > 0:` เพื่อให้สั่งทำงาน `send_line_message()` เฉพาะตอนมีเทสพังเท่านั้น

## 3. ดึงคำค้นหาจากไฟล์ Excel/CSV (Data-Driven with File) 📊
- **แนวคิด:** แยกข้อมูลที่ใช้ทดสอบออกจากโค้ด เพื่อให้ทีมงานคนอื่น (เช่น QA หรือ Admin) สามารถเพิ่มคำทดสอบได้เองผ่านไฟล์ Excel/CSV
- **สิ่งที่ต้องทำ:**
  - สร้างไฟล์ `test_data.csv`
  - ใช้ไลบรารี `pandas` หรือโมดูล `csv` ใน Python เพื่ออ่านข้อมูลมาป้อนให้กับ `@pytest.mark.parametrize` ใน `test_court_ui.py`

## 4. ทดสอบข้ามเบราว์เซอร์และอุปกรณ์มือถือแบบเต็มรูปแบบ (Cross-Browser & Mobile Testing) 📱
- **แนวคิด:** ปัจจุบันมีการตั้งค่า fixture สำหรับ iPhone 13 ไว้แล้ว แต่ยังไม่ได้ใช้อย่างเต็มที่
- **สิ่งที่ต้องทำ:**
  - เพิ่มการรันเทสบนหน้าจอขนาดเล็ก (Mobile Viewport) สำหรับทุกๆ ฟีเจอร์ เพื่อให้มั่นใจว่าเว็บรองรับ Responsive Design ได้ 100%
